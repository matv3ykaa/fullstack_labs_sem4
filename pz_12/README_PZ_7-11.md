# Фронтенд и бэкенд разработка. Практические занятия 7-11

## Стек

**Бэкенд:** Node.js, Express 5, bcrypt, jsonwebtoken, nanoid, cookie-parser, swagger-jsdoc, swagger-ui-express

**Фронтенд:** React 18, React Router, axios, SCSS

---

## ПЗ 7. Базовая аутентификация + CRUD товаров

Написан Express-сервер с регистрацией и входом пользователей. Пароли хешируются через bcrypt. Реализованы маршруты для работы с товарами (создание, получение списка, получение по id, обновление, удаление). Пользователь: `id`, `email`, `first_name`, `last_name`, `password`. Товар: `id`, `title`, `category`, `description`, `price`. Документация через Swagger.

## ПЗ 8. JWT-авторизация

В сервер из ПЗ 7 добавлена выдача JWT access-токена при входе. Написан `authMiddleware` для проверки токена из заголовка `Authorization: Bearer <token>`. Защищены маршруты `GET/PUT/DELETE /api/products/:id`. Добавлен маршрут `GET /api/auth/me`, который возвращает текущего пользователя по токену.

## ПЗ 9. Refresh-токены

Добавлены refresh-токены с отдельным секретом и временем жизни 7 дней. При входе сервер выдаёт оба токена. Реализована ротация: `POST /api/auth/refresh` удаляет старый refresh-токен и выдаёт новую пару. Хранилище токенов в памяти через `Set`.

## ПЗ 10. Хранение токенов на фронтенде

Написана React-часть приложения. Access-токен хранится в `localStorage`, refresh-токен хранится в `HttpOnly cookie` (сервер устанавливает его через `res.cookie`). Настроены axios interceptors: request-перехватчик подставляет токен в каждый запрос, response-перехватчик при 401 автоматически запрашивает новый access-токен и повторяет исходный запрос. Реализованы страницы входа, регистрации, каталога товаров и детальной страницы товара. Есть модальное окно для создания и редактирования товаров.

## ПЗ 11. RBAC (роли пользователей)

Добавлена система ролей: `user`, `seller`, `admin`. Написан `roleMiddleware`. Права доступа:

- гость: регистрация и вход
- пользователь: просмотр товаров, `GET /api/auth/me`
- продавец: создание и редактирование товаров
- администратор: удаление товаров, управление пользователями (`GET/PUT/DELETE /api/users/:id`)

На фронтенде UI адаптирован под роль: кнопки добавления/редактирования/удаления показываются только нужным ролям. Добавлена страница пользователей для администратора с возможностью менять роль, блокировать и разблокировать пользователей. При истечении сессии показывается модальное окно с предложением войти заново.

## Запуск

```bash
# Бэкенд
cd pz_11/backend
npm install
node index.js
```

```bash
# Фронтенд
cd pz_11/frontend
npm install
npm start
```

Бэкенд запускается на `http://localhost:3000`, фронтенд на `http://localhost:3001`.

Swagger UI доступен по адресу `http://localhost:3000/api-docs`.

При первом запуске автоматически создаётся аккаунт администратора: `admin@shop.ru` / `admin123`.
