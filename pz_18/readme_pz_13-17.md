# Описание работы

В рамках занятий 13–17 реализовано веб-приложение для тасков **flow**. Это менеджер тасков с заметками. Приложение развивалось итеративно, то есть каждое практическое занятие добавляло новый слой функциональности поверх предыдущего.

---

## Итоговая структура проекта, реализованная в ПЗ 17

```text
flow/
├── content/
│   ├── home.html        # Динамический контент главной страницы
│   └── about.html       # Страница «О приложении»
├── icons/
│   ├── favicon.ico
│   ├── favicon-16x16.png
│   ├── favicon-32x32.png
│   ├── icon-192x192.png
│   └── icon-512x512.png
├── index.html           # App Shell
├── app.js               # Клиентская логика
├── style.css
├── manifest.json        # Web App Manifest
├── sw.js                # Service Worker
├── server.js            # Node.js сервер (Express + Socket.io + web-push)
└── package.json
```

**Клиент:** HTML, CSS, JavaScript, Service Worker, Web App Manifest, Socket.IO (клиент), PushManager  
**Сервер:** Node.js, Express, Socket.IO, web-push, body-parser, cors

## Что сделано по практическимзанятиям

### ПЗ 13 (Service Worker)

Реализован базовый менеджер задач с офлайн-доступом. Service Worker кэширует статические ресурсы при установке (стратегия Cache First) и отдаёт их при отсутствии сети. Данные задач хранятся в `localStorage`. Дополнительно реализованы: отметка задач выполненными, модальное окно для добавления заметки к задаче.

### ПЗ 14 (Web App Manifest)

К приложению добавлен `manifest.json` с полями `name`, `short_name`, `start_url`, `display`, `background_color`, `theme_color`, `description`, `orientation`, `icons`. Подготовлены иконки нескольких размеров (16×16 до 512×512). В `index.html` подключены манифест и мета-теги для Android и iOS. Service Worker обновлён до версии `v2` (теперь кэшируются и иконки, и файл манифеста). Приложение устанавливается как PWA.

### ПЗ 15 (HTTPS + App Shell)

Приложение переведено на архитектуру App Shell, это статический каркас (шапка, навигация, футер), который кэшируется при установке, динамический контент (`/content/*.html`) подгружается через `fetch` по стратегии Network First с фолбеком на кэш. Добавлена навигация между страницами "Главная" и "О приложении" без перезагрузки. Для локального HTTPS использована утилита `mkcert` + `http-server --ssl`.

### ПЗ 16 (WebSocket + Push)

Добавлен Node.js-сервер на Express. Реализована двусторонняя связь через Socket.IO (при добавлении задачи на одной вкладке остальные получают всплывающее уведомление). Реализована подписка на push-уведомления через `PushManager` и `web-push` с VAPID-ключами. В Service Worker добавлен обработчик события `push`, он показывает системное уведомление даже при закрытом приложении. В интерфейс добавлены кнопки включения/отключения уведомлений.

### ПЗ 17 (напоминания с отложенным Push)

К приложению добавлен раздел заметок с напоминаниями. Пользователь может указать дату и время, и тогда сервер ставит `setTimeout` и отправляет push точно в нужный момент. В уведомлении есть кнопка "Отложить на 5 минут", клик по ней отправляет запрос на сервер (`POST /snooze`), который отменяет старый таймер и запускает новый. Обработка клика по кнопке уведомления реализована в Service Worker через событие `notificationclick`.
